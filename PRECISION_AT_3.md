# Оценка качества поиска — Precision@3 (QA-05)

## Методология

Для 10 эталонных поисковых запросов проверяется, попадает ли ожидаемый (релевантный) документ в топ-3 результатов выдачи `GET /api/v1/search?q=...`.

**Precision@3** для одного запроса = 1, если ожидаемый документ найден среди первых 3 результатов, иначе 0.
**Итоговая метрика** = (сумма Precision@3 по всем запросам) / (количество запросов).

## Как заполнить таблицу

1. Убедитесь, что в системе загружено несколько документов (можно использовать `init.sh` для загрузки 10 тестовых PDF + свои документы).
2. Для каждого запроса выполните поиск через Swagger UI (`/docs`) или интерфейс на `http://localhost:5173`.
3. Посмотрите на топ-3 результата и отметьте, встретился ли среди них ожидаемый документ.
4. Впишите `Да`/`Нет` в столбец «Найден в топ-3» и посчитайте итоговую метрику внизу.
