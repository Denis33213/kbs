# Оценка качества поиска — Precision@3 (QA-05)

## Методология

Для 10 эталонных поисковых запросов проверялось, попадает ли ожидаемый (релевантный) документ в топ-3 результатов выдачи `GET /api/v1/search?q=...`.

**Precision@3** для одного запроса = 1, если ожидаемый документ найден среди первых 3 результатов, иначе 0.
**Итоговая метрика** = (сумма Precision@3 по всем запросам) / (количество запросов).

Оценка проведена на корпусе из 7 разнородных документов, загруженных через интерфейс системы: учебные материалы, лабораторные работы, служебные документы.

## Анализ промахов

Запросы №5 и №6 не нашли ожидаемый документ. Оба документа (`Interaktivnaya-Vizualizaciya-...pdf` и `ЗАДАНИЯ БПИ 2403...pdf`) при загрузке проиндексировались всего в **1 текстовый фрагмент** каждый — заметно меньше, чем остальные документы (9–45 фрагментов). Вероятная причина: эти PDF содержат преимущественно изображения/сканы без полноценного текстового слоя, из-за чего `pdfplumber` извлекает крайне мало текста, и релевантных совпадений для тематических запросов недостаточно для попадания в топ-3.

Это ожидаемое ограничение текущей реализации (нет OCP/OCR для сканированных документов) и не является дефектом самого алгоритма ранжирования — на документах с нормальным текстовым слоем (8 из 10 запросов) система стабильно находит релевантный документ на первой позиции.
